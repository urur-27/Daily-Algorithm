# 🧩 Find Pivot Index

**난이도:** Easy  
**플랫폼:** LeetCode  
**유형:** Array, Prefix Sum  
**링크:** https://leetcode.com/problems/find-pivot-index/

---

## 📝 문제 설명

정수 배열 `nums`가 주어질 때,  
배열에서 피벗 인덱스(pivot index)를 반환하라.

피벗 인덱스란,
- 해당 인덱스의 **왼쪽 원소들의 합**
- 해당 인덱스의 **오른쪽 원소들의 합**
  이 **서로 같은 위치**를 의미한다.

만약 피벗 인덱스가 없다면 `-1`을 반환한다.  
여러 개라면 **가장 왼쪽**에 있는 피벗 인덱스를 반환한다.

---

## 🔍 예시

입력  
nums = [1,7,3,6,5,6]

출력  
3

설명
- 왼쪽 합: 1 + 7 + 3 = 11
- 오른쪽 합: 5 + 6 = 11

---

입력  
nums = [1,2,3]

출력  
-1

---

입력  
nums = [2,1,-1]

출력  
0

설명  
왼쪽에는 아무 원소도 없으므로 합은 0,  
오른쪽 합은 1 + (-1) = 0이다.

---

## 📋 제약 조건 (Constraints)

- `1 <= nums.length <= 10^4`
- `-1000 <= nums[i] <= 1000`

---

## 💡 접근 방식

### 핵심 아이디어
- 전체 배열의 합을 먼저 구한다.
- 순회하면서 현재 인덱스를 기준으로:
    - 왼쪽까지의 합 * 2 + 현재 인덱스의 값 = 전체 값(이 조건을 만족시키는 인덱스가 리턴값)
