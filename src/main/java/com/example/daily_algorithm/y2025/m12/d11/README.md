# 🧩 Happy Number

**난이도:** Easy  
**유형:** Hash Table, Two Pointers, Math  
**링크:** https://leetcode.com/problems/happy-number/

---

## 📝 문제 설명

정수 `n`이 주어졌을 때, 아래 과정을 반복한다:

1. 숫자의 각 자릿수를 제곱한 값의 합을 구한다.
2. 그 결과를 새로운 숫자로 바꾼 뒤 다시 같은 작업을 반복한다.

이 과정이 **언젠가 1에 도달하면 그 수는 “Happy Number”**,  
1에 도달하지 못하고 **같은 값이 계속 반복되면 “Happy Number가 아님”** 으로 판단한다.

예:
- 19 → 1² + 9² = 82
- 82 → 8² + 2² = 68
- ... → 계속 반복하다 1이 되면 Happy Number

핵심은 “같은 수가 반복해서 나오면 사이클(cycle) 발생” 이다.

---

## 🔍 예시

**예시 1**
- 입력: `n = 19`
- 출력: `true`
- 이유: 변환 과정을 반복하면 결국 1에 도달

**예시 2**
- 입력: `n = 2`
- 출력: `false`
- 이유: 1에 도달하지 못하고 값들이 반복됨 (cycle 발생)

---

## 📋 제약 조건 (Constraints)

- `1 <= n <= 2^31 - 1`

---

## 💡 접근 방식

### **HashSet으로 사이클(cycle) 존재 여부 탐지**

Happy Number의 핵심은:

- 1에 도달하면 **Happy Number → true**
- 1에 도달하지 못하고 **같은 숫자가 반복되면 cycle → false**

따라서 HashSet을 사용해 이미 등장했던 숫자를 기록하면 된다.

### 알고리즘 개요

1. HashSet 생성
2. 무한 루프 시작
3. 현재 숫자가:
    - **1이면** → true 반환
    - **Set에 이미 있다면** → cycle 발생 → false 반환
4. 현재 숫자를 Set에 추가
5. 숫자를 "각 자릿수 제곱의 합"으로 변환
6. 반복

### 시간/공간 복잡도

- 시간: **O(log n)**  
  (자릿수 연산 반복, n이 계속 줄어듦)
- 공간: **O(log n)**  
  (Set에 저장되는 중간 숫자 개수는 매우 적음)
